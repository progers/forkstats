In 2013 the browser wars sprouted a new rendering engine when Blink forked from Webkit. At the time of the fork, Webkit had 1.8 million lines of C++, 2,500 commits per month and was the most popular engine. On mobile, Webkit backed the top 3 browsers (Apple Safari, Google Chrome, Android Browser), accounting for over 75% of mobile browsers. This post is a look at the Blink/Webkit fork one year later: how have the projects diverged, who is driving them, and where are they headed?

What is a browser engine?

Browser engines load, layout, and display webpages. The Webkit project includes more than just a browser engine: it also contains two application layers (Webkit1 and the multiprocess Webkit2), a Javascript engine (JavascriptCore), 30,000 tests, etc. At the time of the fork there were 6 distinct build systems. For Blink, both the application layer (Chromium) and javascript engine (V8) live in separate repositories. In this analysis, "Source/" is the C++ code of the WebKit project (including JavascriptCore and Webkit2) and "core" is just the common code shared by Blink and Webkit such as Source/WebCore and Source/WTF. All code behind the data here is open source.
